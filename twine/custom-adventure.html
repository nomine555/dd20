# INTRODUCCIÓN Este tutorial te guiará por los pasos necesarios para crear tu propia aventura para el sistema *GameThrough* de Digital d20 Adventures. *GameThrough* utiliza Twine para crear aventuras con elementos enlazados y le aporta además mapas interactivos, música y la biblioteca de contenido SRD5. Para crear una aventura solo tienes que escribirla con el formato adecuado y observar los resultados, así: <a href="https://digitald20.com/wp-content/uploads/2020/04/ejemplo-markdown.jpg" target="_blank" rel="noreferrer">![large](https://digitald20.com/wp-content/uploads/2020/04/ejemplo-markdown.jpg)</a> Con todo esto, generaremos una aventura que podremos cargar en Digital d20. <a href="https://digitald20.com/play/custom-adventure/fan-adventure.html" download>Descarga la aventura de ejemplo</a> y ábrela con <a href="https://twinery.org/" target="_blank" rel="noreferrer">Twine</a>. ## Twine Antes de empezar, necesitarás la herramienta gratuita Twine. Utilizar Twine es muy sencillo, y no requiere ningún conocimiento previo ni nociones de programación. Descarga Twine <a href="https://twinery.org/" target="_blank" rel="noreferrer">aquí</a> o utiliza su versión online. Twine organiza las escenas en tarjetas llamadas "Pasajes" que puedes editar y reordenar. Visita la sección [[Twine]] para más información. <a href="https://digitald20.com/wp-content/uploads/2020/04/flow.jpg" target="_blank" rel="noreferrer">![large](https://digitald20.com/wp-content/uploads/2020/04/flow.jpg)</a> ## Markdown Markdown es un sistema de codificación que permite escribir texto enriquecido mediante texto plano. No tienes que descargar nada para escribir con Markdown, puesto que Markdown es *una forma de escribir*. Es muy sencillo, te damos todas las claves en la sección *Markdown*. ## Sistema GameTrough El sistema *GameTrough* de Digital d20 utiliza Markdown. Apoyándonos en Markdown y en construcciones propias, el sistema *GameTrough* puede interpretar el código para: * Generar mapas para navegar entre las diferentes localizaciones. * Incorporar el SRD5 completo e hiperenlazar todas las palabras clave. * Mostrar información de conjuros, objetos mágicos y monstruos allá donde aparezcan. * Incluir música. * Mostrar un diagrama de flujo con el desarrollo de la aventura que lleve a cada encuentro. Crear una aventura utilizando el sistema *GameTrough* es tan sencillo como crear las escenas en *Pasajes* de Twine y luego cargarla en Digital d20. Encontrarás más información sobre Twine en la sección *Twine*. ## Cargar una aventura Una vez tengas al aventura lista, puedes cargar el archivo HTML creado por Twine en la zona superior de esta página. Recuerda que subir este archivo cargará la aventura nueva. Para volver al tutorial deberás pulsar *Clear*. # INDICE * [[Introducción]] * [[Markdown]] * [[Pasajes]] * [[Encuentros]] * [[Enlaces]] * [[Contenido SRD5 enlazable]] * [[Contenidos multimedia]] * [[Monstruos]] * [[Mapas]] * [[PNJs]] * [[Diagrama de flujo]] * Herramientas adicionales * [[Twine]] * [[Posicionador de etiquetas]] * [[Alojamiento de archivos online]] * [[Herramientas para mapas]] * [[Homebrewery]] * [[Herramientas de edición]] # PASAJES Un pasaje es un elemento en Twine. Es la forma en que Twine organiza las escenas de una historia. El sistema *GameThrough* utiliza los pasajes para crear tres tipos de escenas: * **Escenas tipo índice:** Que mantienen el índice en la parte izquierda, con información únicamente en la derecha. * *Encuentros*: Escenas que incorporan música, imágenes y/o mapas e información adicional enla zona izquierda. * **Lugares:** Mapas navegables con descripciones emergentes. <a href="https://digitald20.com/wp-content/uploads/2020/04/ejemplo-pasaje.jpg" target="_blank" rel="noreferrer">![large](https://digitald20.com/wp-content/uploads/2020/04/ejemplo-pasaje.jpg)</a> Los pasajes se componen de: | Contenido | Descripción | |:----:|:-------------| | Título | El **Título** es el nombre del pasaje, por el cual será referenciado allá donde aparezca. | | Etiquetas | Las **Etiquetas** están situadas entre el título y el texto y aportan funcionalidades adicionales, como veremos a continuación. | | Texto | El **Texto** es el contenido del pasaje propiamente dicho. | Pulsando en Twine el botón inferior **+Pasaje** se crea un pasaje. ## Etiquetas para pasajes Una vez dentro de un pasaje, pulsar en **+Etiqueta** nos permite añadir distintos comportamientos a cada pasaje, según: | Contenido | Descripción | |:----:|:-------------| | <div class="codigo">flow</div> | Hace que el pasaje sea visible en el *Diagrama de flujo*. Esto no afecta a otros comportamientos. | | <div class="codigo">encounter</div> | El pasaje es un encuentro, con una o varias imágenes, música y mapas. Además tiene sección **A la vista** y **Conclusión**. Para más detalles ver sección *Encuentros*. | | <div class="codigo">@main-map</div> | De todos los mapas (ver sección [[Mapas]]) uno de ellos ha de ser el inicial, el primero que se vea al pulsar en la sección apropiada. Ese pasaje tiene que tener la etiqueta **@main-map**. | | <div class="codigo">@monsters</div> | Las estadísticas de los monstruos de la aventura estarán todas en este pasaje. | | <div class="codigo">@index-main</div> | Para el índice de la aventura. El resto de pasajes serán enlazados desde aquí. | | <div class="codigo">@resumen</div> | El primer contenido que muestre la aventura será el resumen. Este pasaje no suele tener imágenes (no es encuentro). | | <div class="codigo">@npcs</div> | Pasaje que recoje los PNJ's de la aventura. | | <div class="codigo">@notes</div> | Las notas al margen de la aventura estarán todas en este pasaje. | ## Encuentros Los encuentros son un tipo especial de pasaje que incluyen una zona izquierda con imágenes, música, mapas e información adicional. Puedes aprender más sobre esto en la sección *Encuentros*. # ENLACES ## Enlaces a contenido liberado del SRD5 El sistema *GameThrough* genera automáticamente enlaces para las palabras clave que se desee (y formen parte del SRD5). De este modo se pueden generar enlaces para: * Conjuros * Monstruos * Reglas * Objetos mágicos existentes en el SRD5, que se generarán de forma automática para aparecer en la zona de información izquierda. <a href="https://digitald20.com/wp-content/uploads/2020/04/notas-al-margen.jpg" target="_blank" rel="noreferrer">![ancho50](https://digitald20.com/wp-content/uploads/2020/04/notas-al-margen.jpg)</a> Para ello, hay que escribir el texto *entre asteriscos*. El sistema detectará el contenido del texto destacado, y creará un hipervínculo con la entrada correspondiente para mostrar un *objeto mágico*, *trampa*, etc. Aquí puedes ver todo el material SRD5 que puedes enlazar: [[Contenido SRD5 enlazable]]. ## Enlaces a contenido propio Se puede utilizar también este sistema para crear hipervínculos automáticamente a contenido propio de la aventura. De este modo se pueden generar enlaces para: * **Monstruos:** Se mostrarán siempre en la zona izquierda de un encuentro, con una cruz para cerrarlos una vez se hayan consultado. * **PNJ's:** Tienen una sección propia, aunque también se pueden mostrar en la sección izquierda, si se crea una entrada con su nombre. * **Pasaje:** Un enlace a un pasaje (creado poniendo el nombre del pasaje entre asteriscos) llevará al usuario a dicho pasaje. * **Notas al margen:** Las notas al margen se mostrarán siempre en una ventana flotante. * **Mapas:** Dentro de un mapa los enlaces pueden ser de dos tipos: * **Enalces entre mapas** (textos en verde) navegan entre los distintos mapas de la aventura. * **Puntos de interés** (círculos veis) proporcionan información adicional de una zona concreta de un mapa. Para ello, además de escribir el texto \*entre asteriscos\*, debe existir la entrada correspondiente como título de primer nivel en los pasajes **@monsters**, **@notes** y **@main-npcs**. El sistema detectará el contenido del texto destacado, y creará un hipervínculo con la sección correspondiente. Los **Monstruos** se muestran en la zona de información izquierda. Los **PNJ's** tienen una sección propia, aunque también se pueden mostrar en la zona de información izquierda si se crea un enlace con su nombre. Las **Notas al margen** se muestran en una ventana flotante. # CONTENIDOS MULTIMEDIA Hay cinco tipos de contenido multimedia que se pueden añadir a las aventuras. * Las imágenes y los audios se pueden poner en cualquier sitio en cualquier tipo de entrada. * La música y los mapas solo se admiten en los pasajes tipo Encuentro. ## Audio Los archivos de audio aparecerán al inicio del pasaje con un minireproductor. Se tienen que introducir **antes** de cualquier otro contenido en el pasaje, con el siguiente formato: <div class="cuadro codigo">\!\[audio](www.rutadelaudio.mp3)</div> ## Imágenes en encuentros **En los pasajes de tipo encuentro** se acompañará el texto con entre una y cuatro imágenes. Estas aparecerán en la zona izquierda del encuentro y, al pulsar sobre cada una de ellas, se verá en una ventana flotante, a pantalla completa. Se tienen que introducir **antes** de cualquier otro contenido en el pasaje, con el siguiente formato: <div class="codigo cuadro">\!\[image](www.rutadelaimagen.jpg)</div> Se soporta cualquier tipo de archivo. ## Imágenes incrustadas En cualquier tipo de pasaje se pueden incluir imágenes intercaladas en cualquier parte del pasaje. Hay tres formatos para hacerlo, dependiendo del tamaño en que se quiera mostrar: <div class="cuadro codigo"> \!\[small](www.rutadelaimagen.jpg) <br/> \!\[medium](www.rutadelaimagen.jpg) <br/> \!\[large](www.rutadelaimagen.jpg) </div> Se soportan archivos tipo jpg y png. # MONSTRUOS Los monstruos se enlazan igual que el resto de contenidos, poniendo el nombre del monstruo \*entre asteriscos\*. Si el monstruo forma parte del SRD5 se mostrará el monstruo de las reglas. Si es un monstruo creado para la aventura, se crea el enlace de igual modo, pero hay que incluir el monstruo en el pasaje @monsters con un formato específico. Cuando se pulsa sobre el enlace de un monstruo, este se muestra en la zona de información izquierda. A continuación tienes un monstruo de ejemplo. Puedes pinchar enla imagen para verla más grande o, si tienes dudas, consulta el código después de la imagen. <a href="https://digitald20.com/wp-content/uploads/2020/04/ejemplo-monstruo.jpg" target="_blank" rel="noreferrer">![large](https://digitald20.com/wp-content/uploads/2020/04/ejemplo-monstruo.jpg)</a> En la aventura de ejemplo <a href="https://digitald20.com/play/custom-adventure/fan-adventure.html" download>Secuestro en Brookmili</a> tienes monstruos de ejemplo para que puedas utilizar su formato. Además, en el siguiente archivo tienes todo el contenido del SRD (incluidos los monstruos) por si quieres utilizar el formato de alguno como base para crear los tuyos propios: <a href="https://digitald20.com/wp-content/uploads/2020/04/mardkdown-srd5.txt" target="_blank" rel="noreferrer">Biblioteca de monstruos SRD5</a> # MAPAS Tanto los mapas de navegación como los que se incluyen en los pasajes de tipo encuentro se generan del mismo modo. Un pasaje de tipo mapa puede ser incluido en cualquiera de las dos secciones. <a href="https://digitald20.com/wp-content/uploads/2020/04/ejemplo-mapa-e1586858334706.jpg" target="_blank" rel="noreferrer">![large](https://digitald20.com/wp-content/uploads/2020/04/ejemplo-mapa-e1586858334706.jpg)</a> ## Estructura Ha de incluirse, el inicio del pasaje, la imagen del mapa: <div class="cuadro codigo">\!\[map](www.rutadelaimagen.jpg)</div> Con eso simplemente ya se mostrará el mapa en la sección correspondiente (recordemos que el primer mapa ha de tener siempre la etiqueta **@main-map**). Además, se pueden añadir dos tipos de enlaces: ### Vínculo a otro mapa Para enlazar desde un mapa a otro únicamente hay que colocar un señalador en el mapa, con el siguiente formato: <div class="cuadro codigo">\# @xxxxxx <\!\-\-20,10--></div> Donde @xxxxxx es la etiqueta del nombre del mapa objetivo, y los dos números son la posición del señalador. En la etiqueta se mostrará el nombre del mapa objetivo. ### Enlace a ventana de información Para mostrar una ventana de información en un mapa hay que colocar un señalador con el siguiente formato: <div class="cuadro codigo">\# Nombre <\!\-\-10,10--> <br/> Texto a mostrar en la ventana flotante. Aquí puede incluirse también un enlace a un pasaje.</div> Donde el nombre es el título a mostrar y la pareja de números es la posición de la etiqueta. El texto se mostrará a continuación del título. ## Posicionador de etiquetas Los puntos X, Y que siguen a los títulos de los mapas son la posicion relativa en porcentaje sobre la imagen de fondo. Tienes la siguiente herramienta a tu disposición para calcularlos fácilmente: [[Posicionador de etiquetas]]. # PNJ's Los personajes no jugadores tienen que agruparse todos en un pasaje con la etiqueta **@main-npcs**. Por lo demás, no se distingue mucho de un pasaje normal. Aunque tienen sección propia, los PNJ's también se pueden mostrar en la zona izquierda de un pasaje. Cada PNJ estará formado por: * **Título de primer nivel:** Será el nombre del personaje, por el que se le referenciará allá donde se escriba entre asteriscos. * **Imagen incrustada (opcional)** Utilizando el formato indicado en [[Contenidos multimedia]] se pueden incluir una o varias imágenes, en cualquier tamaño. * **Texto descriptivo:** El texto que irá, bajo el título con el nombre del personaje. Se puede incluir cualquier referencia a reglas, otros personajes, conjuros, encuentros, etc. * **Ficha de personaje (opcional)** La ficha de personaje con el formato indicado en la sección *Monstruos*. <a href="https://digitald20.com/wp-content/uploads/2020/04/pnjs.jpg" target="_blank" rel="noreferrer">![large](https://digitald20.com/wp-content/uploads/2020/04/pnjs.jpg)</a> # DIAGRAMA DE FLUJO Para el diagrama de flujo de una aventura no es necesario crear pasajes específicos, únicamente hay que agregar la etiqueta **flow** a cada pasaje que queramos que se muestre en el diagrama. Para enlazar pasajes en Twine se han de crear relaciones entre ellos utilizando **\[\[Nombre del pasaje\]\]**. Así, aparecerán en el esquema Twine las flechas de relación. Moviendo los pasajes crearemos desde Twine el diagrama de flujo que se mostrará en la aventura, con la misma disposición de elementos <a href="https://digitald20.com/wp-content/uploads/2020/04/flow.jpg" target="_blank" rel="noreferrer">![large](https://digitald20.com/wp-content/uploads/2020/04/flow.jpg)</a> # MARKDOWN Markdown es un sistema de codificación que permite escribir texto enriquecido mediante texto plano. Las codificaciones propias son muy sencillas: <a href="https://digitald20.com/wp-content/uploads/2020/04/tabla-markdown.jpg" target="_blank" rel="noreferrer"> ![large](https://digitald20.com/wp-content/uploads/2020/04/tabla-markdown.jpg)</a> ## Separación entre líneas * El retorno de carro (tecla ENTER) por sí mismo no produce ningún efecto. * Para crear una nueva línea se utilizan dos espacios al final de una línea. Después de esto, añadir un retorno de carro (tecla ENTER) hará que el espacio sea mayor. <a href="https://digitald20.com/wp-content/uploads/2020/04/lineas.jpg" target="_blank" rel="noreferrer">![large](https://digitald20.com/wp-content/uploads/2020/04/lineas.jpg)</a> # ENCUENTROS Los encuentros son un tipo especial de pasaje que incluyen una zona izquierda con imágenes, música y dos secciones: **A la vista** y **Conclusión**, como se muestra en la imagen: <a href="https://digitald20.com/wp-content/uploads/2020/04/encuentro.jpg" target="_blank" rel="noreferrer">![large](https://digitald20.com/wp-content/uploads/2020/04/encuentro.jpg)</a> Como vemos, la estructura de un pasaje tipo encuentro tiene tres partes: ## Contenido La estructura general del pasaje es igual que la de cualquier otro en cuanto a niveles de títulos, enlaces y demás. ## Contenidos multimedia dentro de encuentros Dentro de un encuentro hay contenido multimedia específico que se muestra de una forma especial en los pasajes de tipo encuentro. Los códigos de introducción han de escribirse al inicio del pasaje para imágenes de encuentro, música, mapas y audio, antes de cualquier otro contenido. * La música, los mapas y las imágenes situadas **antes** del título del encuentro se comportan de forma distinta al contenido multimedia en el resto de pasajes: aparecerán en la zona izquierda del encuentro. Se pueden incluir únicamente una pista de música y hasta cuatro imágenes/mapas. * Las imágenes y los audios **después** del primer titulo en los encuentros se comportan igual que en cualquier otro tipo de Pasaje. De este modo se puede incluir cualquier cantidad de imágenes, pero solo un audio. Puede verse un ejemplo de esto en la imagen al inicio de esta entrada. ## Música **En los pasajes de tipo encuentro** se incluye una música que pueda ambientar la escena. Se tiene que introducir **antes** de cualquier otro contenido en el pasaje, con el siguiente formato: <div class="cuadro codigo">\!\[music](www.rutadelamusica.mp3)</div> La música aparecerá en la zona izquierda, en un botón on/off. ## Mapas **En los pasajes de tipo encuentro** se puede incluir, en la misma zona que las imágenes, uno o varios mapas, que pueden tener información relevante hiperenlazada (ver [[Mapas]]). Pulsar sobre el mapa lo abrirá en una ventana flotante. El código se tiene que introducir **antes** de cualquier otro contenido en el pasaje, con el siguiente formato: <div class="cuadro codigo">\!\[map](@xxxxx)</div> Donde **@xxxxx** es el nombre de la etiqueta que tenga el mapa que enlazamos. Los mapas aparecen en la zona izquierda, junto con las imágenes. ## A la vista y Conclusión En esta zona se muestra lo que los personajes verán en el encuentro. Suelen incluirse las tiradas y conjuros más comunes como *Percepción*, *Investigación* o *Detectar Magia*. <div class="cuadro"> <div class="codigo">AT A GLANCE<br/> ---</div> Texto a mostrar en la zona **A la vista**. </div> <br/> El apartado "Conclusión" se reserva para los apuntes que indiquen los nexos con el siguiente encuentro o qué es necesario que los jugadores hayan visto o hecho antes de pasar al siguiente. Además también es un pequeño resumen para que el DM pueda "hojear" la aventura rápidamente. <div class="cuadro"> <div class="codigo">CONCLUSION<br/> ---</div> Texto a mostrar en la zona **Conclusión**. </div> <br/> # ALOJAMIENTO DE ARCHIVOS ONLINE Para la inclusión de archivos multimedia hace falta subir dichos archivos a algún sitio online. Aquí te ofrecemos algunas opciones sobre dónde hacerlo: ## Dropbox Dropbox es una de las más famosas herramientas online ### Paso 1: Crear cuenta de Dropbox e iniciar sesión Primero nos dirigimos a la web de Dropbox y creamos una nueva cuenta. Dropbox pide nuestros nombres y apellidos, un correo electrónico y una contraseña. Una vez creada la cuenta tendremos 2gb de alojamiento gratuito en la nube. Podemos vincular Dropbox con nuestro Smartphone descargando la aplicación en la AppStore (iPhone) o Play Store (Android), o podemos descargar la versión PC/MacOsX. No es necesario tener nuestra cuenta vinculada con nuestro teléfono u ordenador, pero mejoraremos la productividad y la podremos usar más fácilmente. ### Paso 2: Subir archivos a Dropbox El proceso de subida de archivos a Dropbox para usarlo como servidor de imágenes es muy fácil. Si tenemos Dropbox abierto en el navegador sólo debemos coger y arrastrar los archivos deseados desde nuestro ordenador hasta la ventana de Dropbox y soltarlos. También podemos clickar en el icono de la hoja de papel con un plus para poder añadir uno o más archivos. Si hemos instalado Dropbox en nuestro PC o MacOSx habrá creado una carpeta llamada Dropbox en el escritorio o en Mis Documentos al que podemos arrastrar los archivos para subirlos. Podemos crear carpetas para mantener nuestros archivos organizados. ### Paso 3: Cómo conseguir el link al archivo en el servidor Si vamos a Dropbox.com, clickamos sobre el archivo que queremos usar y vamos a Compartir veremos que nos ofrece “crear un enlace”. Esa URL nos llevará al archivo que hemos subido, pero no a la URL de la carpeta del servidor en la que se encuentra alojado. Si usamos esta primera URL en un archivo HTML como fuente (source o src en HTML) de una etiqueta de imagen no podremos ver el archivo, ya que es necesario tener la dirección de la carpeta del servidor en la que se encuentra alojado. Dropbox no nos facilita esa URL, sólo nos da una URL que nos lleva a la galería y abre esa imagen: https://www.dropbox.com/s/oo4yazg7rwiug3x/imagen-destacada-dropbox.jpg. La URL nos lleva a la imagen, pero si la usamos como src de una imagen no nos servirá. ¿Por qué? Porque al usar Dropbox como host de imagenes nos da la URL a una galería, no al archivo de imagen nativo. Existe una manera de modificar la URL y poder apuntar directamente al archivo alojado en el servidor. Para eso, únicamente debemos cambiar https://www.dropbox.com/ por https://dl.dropboxusercontent.com/. Nuestra nueva URL sería: https://dl.dropboxusercontent.com/s/oo4yazg7rwiug3x/imagen-destacada-dropbox.jpg. Esta URL apunta directamente a la imagen, sin más código HTML, por lo que la podemos usar en nuestras firmas de email o webs sin miedo a que caduquen y nos las borren. ## Sube Fotos .com Sube Fotos es un *site* gratuito de alojamiento en la nube que permite cargar archivos de imágenes y proporciona enlaces de acceso a los archivos. Puedes ir a Sube Fotos desde este enlace: https://subefotos.com ### Paso 1 Elije la foto que quieras subir. ### Paso 2 Elije el formato de redimensión o "no redimensionar". ### Paso 3 Elije si quieres que el archivo sea público o no. ### Paso 4 Cuando la imagen se cargue, aparecerán los enlaces de acceso directo al archivo, que son los que podrás utilizar en Twine para cargar la imagen # TWINE Twine es una herramienta digital que permite generar con mucha facilidad redes de textos interrelacionados.
